DNS: Request and parse DNS resource records

Jan 2011 - Apr 2011, The George Washington University, Washington DC

This is the hands-on work for my class at The George Washington University. I studied DNS-related protocols and as a practice of programming I designed and implemented a set of C++ classes used to request and parse resource records from DNS server.

A request packet is sent to DNS server, then the result is returned in a response packet. Current implementation only supports sending message with UDP protocol, and only parses resource records with types of A, CNAME, and TXT, although other types of resource records are also fetched with response packet.Please find source code and documents at Github.

DNS protocol defines a unified packet format for request and response message. Plese find some discussions here about DNS protocols.

Technical Summary

Languages: C++
Edit, Compile, and Debug: Mac OS X, Xcode 4
Techniques: STL, socket, UDP, DNS protocol and message format

Hat: Simple management for personal photos

Jan 2011 - Apr 2011, The George Washington University, Washington DC

This is the hands-on work for my class at The George Washington University. The idea is using a central management system to manage our personal photos distributed on heterogenous storage. Some kinds of characteristic data of photos, such as file stat, meta data, hash, and histogram data, are extracted or calculated, and then collected into a central database. Functions like complex searching and finding identical or similar photos are realized based on database query and analysis of characteristic data. Other advanced functions then may be possible, such as managing personal photos on Facebook and Flickr, automatic deduplication and duplication of photos for backup, and so on.

The concept is demonstrated with a prototype design of software architecture and three simple command line programs. One is used for database update, data query, and analysis. One is used for monitoring local file system, detecting changes of photo files, extracting photo's characteristic data, updating records in database. The other is a user interface program. The programs are developed on Mac OS X system with standard C++. Three programs communicate to each other based on Zeroc Ice, a Internet communication middle-ware. Other libraries including OpenCV, ImageMagik, and OTL are used to simplify the development. Please find source code and documents at Github.

Technical Summary

Languages and libraries: C++, STL, SQL, OTL
Operating system and database: Mac OS X 10.6, SQLite 3
Edit, Compile, and Debug: Xcode 4
Network/ORB framework: Zero Ice 3.3
Image processing libraries: OpenCV, ImageMagik
Characteristic data analysis File stat, Meta data, Hash, Histogram, Euclidean distance

Tie: Share Moods, Share Today!

Sep 2010 - Dec 2010, The George Washington University, Washington DC

This is the hands-on work for my class at The George Washington University. The idea is exploring a typical architecture of social network applications. A GUI application running on mobile devices is used to collect users' daily information like current mood, and then transfer it into a central service system. The service calculates and feeds some statistic information in a long term basis back to users. Sharing can be realized on friends-based relationship. Advanced functions like diagnosis of personal life, making friends, and automatic greeting are possible.

As a proof of concept, a prototype architecture is designed and a simplified software system is demonstrated. Client software is developed on Android system. Service system is developed on popular LAMP (Linux, Apache, MySQL, PHP) platform. Client and server communicate with REST-style or similar interface. Please find source code and documents at Github.

Technical Summary

Languages and Marks: Java, PHP, SQL, XML, HTML/CSS
Operating systems or platforms: Android, LAMP (Linux, Apache, MySQL, PHP)
Database and libraries: MySQL, OTL
Other Techniques: HTTP, Web Service
