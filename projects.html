PPEngine: Towards a open peer-to-peer network

Jan 2010 - present, The George Washington University, Washington DC

Academia and industry had studied peer-to-peer (P2P) networks for over 10 years and a large amount of applications were developed and deployed. However, many factors had hindered the extensive application of P2P networks, in which include the issues like security and interoperability. Most of current P2P overlay networks are designed and implemented as parts of applications and solved problems within a single application domain. Some general P2P frameworks like JXTA were developed but have not obtained extensive confirmation. Based on the experience of P2P system development, I think some problems may be solved if we have not discussed P2P overlay network only within certain application domains, but from the viewpoints of a network architecture and related protocols. I did the original work of this idea when I took classes of Computer Network and Network Security at The George Washington University in spring 2010 when I was a master student, and extended the work during the following classes.

  • Design and define a open P2P network architecture to support interoperability.
  • Discuss the universal description of contents in P2P networks.
  • Discuss the interface between peers.
  • Discuss the algorithms for topology of P2P overlay network.

Technical summary

  • Linux, Windows
  • C++, STL
  • Winsock, Berkeley sockeet
