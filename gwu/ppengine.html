PPEngine: Refactoring of a peer-to-peer video streaming system

May 2010 - Apr 2011, The George Washington University, Washington DC

This is a hands-on project for my coursework at The George Washington University, I did refactoring of a peer-to-peer (P2P) video streaming system on code, design, and architecture level.

Systems Architecture

PPIO is a software framework to ease the programming of network communications involving devices behind NAT. A service system (a single server or a distributed network) consists of devices in public domain (with public IP addresses) is used as a helper to make devices behind NAT reachable. The procedure to establish peer-to-peer connections is wrapped in PPIO.

PPStream is designed as a local helper service that downloads video contents with multiple ways to ensure playback quality. The video data (RTP packets) is stored in a buffer first and then fed to the player with RTSP protocol or alternatives. Different from traditional stream media system, in which all clients connect to the dedicated server and download media contents, PPStream works in a peer-to-peer fashion. At most cases, PPStream nodes connect to each other directly, exchange their buffering information, and propagate video contents as needed. Accessing server occurs only if the data is urgent as to a time window of playback. A PPStream node keeps track on a small sub-set of all nodes in the system, organizes them into three layers of Members, Neighbors, and Partners according to the possibility of sharing data with them. Buffering information of a node is packeted into a message and propagated among nodes with a Gossip-like protocol that is optimized for media streaming applications. Please find source code and documents at Github.
