PPEngine: Refactoring of a peer-to-peer video streaming system

Jan 2010 - Apr 2011, The George Washington University, Washington DC

This is a hands-on project for my coursework at The George Washington University. I did some refactoring on a peer-to-peer video streaming system. The aim of this work is not to work out an application system with full functions, rather to practice software refactoring and discuss using design patterns in refactoring.

ActiveX control is a component on Windows system with binary compatible interface. Video decoder vendor usually provides a ActiveX control as a wrapper of the functions of a video player. In addition, flash is more and more used as a GUI element recently. There is also an ActiveX control distributed with adobe flash player to support flash playback. Here I demonstrate a media player framework based on multiple ActiveX controls. It supports multiple video formats based on the ActiveX controls distributed with Windows Media Player and Real Player (as well ass other media players). Its entire GUI (all visible elements) is realized with a flash. Please find codes and documents at:
https://github.com/maoxuli/fwplayer

There are many formats of video encoding and packaging. We must parse a video format before we can handle the video contents. With object-oriented design and some design patterns we can establish a class structure that is easy to extend to support parsing a new video format. Please find codes and documents at:
https://github.com/maoxuli/msplit

RTP/RTSP protocols are widely used in media streaming applications. RTSP is a HTTP extension. It works as a session management protocol in media streaming applications. RTP/RTCP protocols are usually used to transfer real-time media contents. Please find source code and documents at:
https://github.com/ppengine

This work is continued as a open-source project after I graduated from The George Washington University. Please find details at:
http://www.ppengine.com
