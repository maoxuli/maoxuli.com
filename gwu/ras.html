Case study: Realtor's Assistant System

Jan 2010 - Apr 2010, The George Washington University, Washington DC

This is a case study for my class of software engineering at The George Washington University. Realtor's Assistant System is taken as a example to practice unified software process and development management. This is a team-based project. My role is a software architect and project manager in this case. Planning and tracking of development process is a large part of my work. We start from requirements analysis, draw up software specification, make software development plan, organize a team and complete a software process. The aim of this project is not working out a complete software system, rather experiencing unified software process. It is not a linear process from analysis to design, to implementation, and to testing. It is a iterative and incremental process consists of a series of workflows including requirements, analysis, design, and implementation. Testing is throughout entire process. and
